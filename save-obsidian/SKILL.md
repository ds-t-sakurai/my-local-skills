# ローカル保存ルール

- **Vault パス（実行環境ごとに使い分ける）**:
  - **WSL / Linux 環境（Claude Code on WSL など）**: `/mnt/c/Users/TakeshiSakurai/Documents/80_Obsidian/DATUM_Valut/`
  - **Windows 環境（Claude Desktop など）**: `C:\Users\TakeshiSakurai\Documents\80_Obsidian\DATUM_Valut\`
  - 上記2つは**同じ実体（Windows 上の同一ディレクトリ）**を指している。実行中の環境に応じて適切な表記を選ぶこと。
  - 判定の目安: `uname` が `Linux` を返す / プラットフォームが `linux` の場合は WSL パス、`win32` などの場合は Windows パス。迷ったら現在の作業ディレクトリ（`pwd`）の形式に合わせる。
  - **混在禁止**: 1つのコマンド・ツール呼び出しの中で WSL パスと Windows パスを混在させない（ファイル書き込みツールはどちらか一方の表記でしか解決できない）。
- **ノート保存先**: 内容に応じて適切なフォルダを選択する
  - `00_Inbox/` — 未整理のメモを一時的に置く場所
  - `10_Daily/` — 日次ノート（Daily Notes）の保存先
  - `20_Notes/` — 学習・思考の永続ノート
  - `30_Projects/` — プロジェクト単位のノート
  - `40_Resources/` — 参考資料・クリッピング
  - `50_1on1/` — 個人ごとの1on1ミーティング記録
  - `Templates/` — テンプレートファイル
- **フォーマット**: Obsidianのfrontmatter（YAML）を必ず付ける
- ノートの作成・編集は **日本語** を基本とする。
- ファイル名は日本語を優先する。スペースは使わず `_` で区切る。内容に適したファイル名にする。

## Obsidian 独自記法（積極的に活用すること）

標準 Markdown だけでなく、Obsidian 独自の記法を活用してノートを読みやすく構造化する。

### 内部リンク・目次

長い文書では先頭に目次を作成し、見出しへジャンプできるようにする。標準 Markdown のアンカーリンク `[text](#anchor)` は **日本語見出しでは Obsidian 上で不安定** なため、**Obsidian の Wiki リンク形式 `[[#見出し名]]`** を使用すること。

| 用途 | 記法 |
|------|------|
| 同一ノート内の見出しへジャンプ | `[[#見出し名]]` |
| 別ノートの見出しへジャンプ | `[[ノート名#見出し名]]` |
| エイリアス（表示文字を変える） | `[[#見出し名\|表示テキスト]]` |
| ブロックリンク（特定段落へ） | `[[#^ブロックID]]` |
| ノート全体の埋め込み | `![[ノート名]]` |
| 見出しセクションの埋め込み | `![[ノート名#見出し名]]` |

**注意**: `[[#見出し名]]` の見出し名は実際の見出しテキスト（`#` 記号を除く部分）と完全一致させる必要がある。記号（`：`、`（）`、`・` など）もそのまま含めること。

### テキストを目立たせる記法

| 用途 | 記法 | 備考 |
|------|------|------|
| 太字（強調） | `**text**` | キーワードに |
| 斜体 | `*text*` | 専門用語・引用語句に |
| 太字＋斜体 | `***text***` | より強い強調 |
| ハイライト（蛍光ペン） | `==text==` | **Obsidian 独自**。黄色背景 |
| 取り消し線 | `~~text~~` | 削除・無効を表す |
| インラインコード | `` `code` `` | コマンド・コード片 |

### Callout（コールアウト）— 最も視覚的に目立つブロック

引用ブロック `>` の先頭行に `[!タイプ]` を書くと、色付きアイコン付きボックスとして表示される。**重要な警告・補足・例などを目立たせる際の第一選択**。

```markdown
> [!note] タイトル（省略可）
> 内容を書く。Markdown 記法もそのまま使える。
> - リストもOK
> - **太字**もOK
```

**主要なタイプ（色・用途で使い分け）:**

| タイプ | 色 | 用途 |
|--------|------|------|
| `note` | 青 | 一般的な注釈 |
| `info` / `summary` / `tldr` / `abstract` | 青 | 情報・要約 |
| `tip` / `hint` / `important` | 緑 | ヒント・重要事項 |
| `success` / `check` / `done` | 緑 | 成功・完了 |
| `question` / `help` / `faq` | 黄 | 質問 |
| `warning` / `caution` / `attention` | 黄 | 警告 |
| `failure` / `fail` / `missing` | 赤 | 失敗・欠落 |
| `danger` / `error` | 赤 | 危険・エラー |
| `bug` | 赤 | バグ |
| `example` | 紫 | 例示 |
| `quote` / `cite` | 灰 | 引用 |

**折りたたみ可能にする:**

- `> [!note]+ タイトル` — デフォルトで開いた状態
- `> [!note]- タイトル` — デフォルトで閉じた状態（長文の補足を畳む際に有用）

### その他の便利な記法

| 用途 | 記法 |
|------|------|
| コメント（表示されない） | `%%text%%` |
| タスクリスト | `- [ ]` / `- [x]` |
| 脚注 | 本文中に `[^1]`、文末に `[^1]: 注釈内容` |
| タグ | `#タグ名` |

## 記法の使い分け方針

- **長い手順書・ドキュメント**: 先頭に `[[#見出し名]]` 形式の目次を必ず付ける
- **重要な警告・注意事項**: `> [!warning]` または `> [!danger]` の Callout を使う（単純な `> ⚠️ 注意：` より目立つ）
- **補足情報・参考リンク**: `> [!info]` または `> [!note]` の Callout
- **例示・コード例**: `> [!example]` の Callout
- **キーワード・用語の強調**: `**太字**` または `==ハイライト==`
- **長い補足を畳んでおきたい**: `> [!note]-` で折りたたみ
- **絵文字での装飾は控えめに** — Callout に組み込まれているアイコンを優先する
